  <WorkProse>
    Golf is inherently a social game, yet the modern digital experience for golfers is surprisingly isolated. Creating a frictionless way for players to find like-minded individuals to hit the links with has been long overdue. The dissatisfaction within the community was clear, scattered across forums, disjointed group chats, and fragmented social media groups.
    
    As Co-Founder and CTO of Links Connect Inc., I set out to build Swingers Golf. <WorkSection>
  <WorkHeader type="ARCHITECTURE" title="Architecting a cross-platform ecosystem." />
  
  <WorkSplitContent subtitle="Speed on the frontend, stability on the backend">
    Building a social network requires an architecture that can support real-time interactions, heavy media storage, and rapid deployment across both iOS and Android simultaneously. A fragmented native codebase was out of the question for our timeline.
    
    I architected a decoupled ecosystem. We leveraged React Native and Expo to maintain a single, highly iterative frontend codebase. For the backend, I designed a robust REST API using PHP and Laravel, ensuring we had a reliable, strictly typed relational data layer capable of handling complex matchmaking algorithms.
  </WorkSplitContent>

  <WorkFeatureList>
    <WorkFeature title="The Mobile Client (Expo & React Native)">
      Built with TypeScript and Tailwind CSS, the frontend delivers a buttery-smooth, native-feeling user interface. Using Expo and Remix allowed for rapid over-the-air updates and shared logic, keeping development cycles incredibly lean.
    </WorkFeature>
    <WorkFeature title="The Core API (Laravel & PHP)">
      The absolute source of truth for the platform. Written in PHP utilizing the Laravel framework, it handles user authentication, connection logic, real-time push notifications, and the core matchmaking engine.
    </WorkFeature>
    <WorkFeature title="Infrastructure (DigitalOcean & AWS)">
      Deployed on DigitalOcean for predictable, scalable compute, coupled with AWS S3 for high-speed, secure media storage. The database layer utilizes MySQL, optimized to maintain high-throughput read/write operations for the social feed.
    </WorkFeature>
  </WorkFeatureList>
</WorkSection>

<WorkSection>

<WorkProse>
  You cannot build a successful sports technology product in a vacuum. The engineering might be
  flawless, but if the user experience doesn't resonate with the actual culture of the game,
  adoption will fail. Instead of a blind public launch, I led a targeted beta testing program with
  over 30 highly experienced golf management professionals. Their industry expertise was invaluable.
  We treated their feedback as strict engineering requirements, utilizing it to rapidly iterate on
  our matchmaking precision, refine our social features, and dial in the overall UX.
</WorkProse>
