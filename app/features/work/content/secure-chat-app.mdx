<WorkSection>
  <WorkHeader type="SECURITY ARCHITECTURE" title="Implementing Kerberos." />

<WorkEvolution
  contextHeading="The standard login vulnerability"
  context={
    <>
      In standard web applications, users often send their credentials directly to the server, which
      validates them and returns a session token. For a highly secure environment, transmitting the
      actual password over the network presents an unnecessary attack vector.
    </>
  }
  evolutionHeading="Ticket-Granting Architecture"
  evolution={
    <>
      To solve this, we implemented a custom Kerberos authentication flow. Instead of verifying
      passwords directly, the system uses a Key Distribution Center (KDC) and Ticket-Granting
      Tickets (TGT). Once a client is authenticated and receives their service ticket, all real-time
      messages are secured end-to-end using AES encryption, ensuring that even if the database is
      intercepted, the chat history remains illegible.
    </>
  }
/>

  <WorkDiagramWrapper caption="Kerberos Authentication Sequence" type="Sequence">
    <Mermaid chart={`
      sequenceDiagram
        autonumber
        participant Client
        participant AS as Auth Server (KDC)
        participant TGS as Ticket Granting Server
        participant Chat as Chat Service

        Client->>AS: Request TGT (User ID)
        AS-->>Client: TGT + Session Key (Encrypted)
        Client->>TGS: Send TGT + Request Chat Service Ticket
        TGS-->>Client: Chat Service Ticket (Encrypted)
        Client->>Chat: Present Ticket + Authenticator
        Chat-->>Client: Mutual Authentication Success
    `} />

  </WorkDiagramWrapper>

<WorkSection>

<WorkFeatureList>
  <WorkFeature title="The Client (React Native)">
    We built a cross-platform mobile application to provide a seamless, native feel for both iOS and
    Android users, handling the local AES encryption and decryption of messages directly on the
    device.
  </WorkFeature>
  <WorkFeature title="The Backend (Python & Flask)">
    Developed the core server logic and Key Distribution Center (KDC) in Python, managing the
    complex cryptographic ticket exchanges and data validation required by the Kerberos protocol.
  </WorkFeature>
  <WorkFeature title="Real-Time Infrastructure (Firebase)">
    Leveraged Firebase to handle the real-time websocket connections, instantly syncing the
    encrypted, unreadable message payloads across all active clients in a chat room.
  </WorkFeature>
</WorkFeatureList>
